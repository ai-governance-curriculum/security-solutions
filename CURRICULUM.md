# Curriculum Guide

This document defines the expected structure for `ai-infra-security-solutions`.

- Track type: solutions
- Paired learning repo: `ai-infra-security-learning`
- Primary content directory: `projects/`

## Layout
- `projects/`: per-project reference implementations
- `modules/`: cross-cutting module solutions (currently delegated to engineer-solutions for security primitives)
- `guides/`, `resources/`: shared material

## Structural Rules
- Project slugs MUST match the paired learning repository.
- Solutions are reference implementations; learners attempt projects first.
